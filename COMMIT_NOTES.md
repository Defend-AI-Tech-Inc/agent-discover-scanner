# MCP Governance — v2.6.0-mcp-governance

## What was built

### New module: `src/agent_discover_scanner/exporters/mcpfw_policy.py`

Public API:
```python
export_mcpfw_policy(scan_result: dict, stance: str = "balanced") -> dict
```

Converts AgentDiscover Scanner MCP findings into a runnable mcpfw policy dict.
Three stances: `strict` (deny-by-default), `balanced` (allow+DLP), `monitor` (log only).

Key behaviors:
- `@modelcontextprotocol/server-everything` always blocked regardless of stance
- `risk: critical` servers always blocked in `balanced` and `strict`
- Unverified + high-risk servers get an `allowed_tools` allowlist in `balanced` and `strict`
- Local scripts get `blocked_tools: [execute_command, shell, eval, ...]`
- Filesystem servers get `allow_paths` restricting reads/writes to scan dir + /tmp
- Database servers get advisory `rate_limit` (mcpfw enforces in v0.2.0)
- Browser/web_search servers get `block_patterns` on argument values
- Global DLP rules (credential detection + prompt injection) always included
- SaaS-specific DLP (Stripe, GitHub, Salesforce, Google, ServiceNow) added when saas detected
- Retry/pagination injection always logged, never blocked (billing amplification attack class)
- Never raises — returns minimal valid stub on any error

### New CLI subcommand: `export-mcpfw-policy`

```bash
agent-discover export-mcpfw-policy <dir> --stance balanced --output policy.yaml
```

Calls `detect_mcp_servers()` directly, no prior scan required.

### New flag on `scan-all`: `--emit-mcpfw-policy`

```bash
agent-discover scan-all . --emit-mcpfw-policy policy.yaml --stance strict
```

Emits policy after scan using MCP findings from the full 4-layer run.

### scan_runner.py changes

- `report["mcp"] = mcp_result` — MCP findings now included in returned report dict
- `report["scan_path"] = str(scan_root)` — scan root included for `allow_paths` generation

### Tests: `tests/test_mcpfw_export.py`

39 tests covering: policy structure, all three stances, always-block packages,
per-capability rules (filesystem/database/code_execution/browser/web_search),
unverified server handling, global DLP, SaaS-specific DLP, deduplication,
network-only detection, YAML validity.

### Demo assets: `demo/mcp-governance/`

- `DESIGN.md` — Phase 1 design: scanner schema, mcpfw schema, mapping table, edge cases
- `README.md` — three-command workflow (discover, review, enforce)
- `sample-policy.yaml` — policy generated by scanning this repo (balanced stance)

## What was NOT changed

- `~/projects/mcpfw/` — read-only reference, no modifications
- Existing scanner detection layers (L1-L4)
- No UI, no daemon changes, no PyPI publish

## How to test

```bash
uv run pytest tests/test_mcpfw_export.py -v    # new tests only
uv run pytest tests/ -v                         # full suite (95 tests)
```

## How to use

```bash
# Standalone discovery + policy generation
agent-discover export-mcpfw-policy /path/to/project --stance balanced

# As part of a full scan
agent-discover scan-all /path/to/project --emit-mcpfw-policy policy.yaml --stance strict

# Enforce
mcpfw wrap --policy policy.yaml -- npx @modelcontextprotocol/server-filesystem /tmp
```
